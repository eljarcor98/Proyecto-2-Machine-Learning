# Guía de Contribución 🤝

¡Bienvenido al equipo! Para mantener el proyecto organizado, sigue estas pautas:

## 🌿 Flujo de Trabajo (Git)

1. **Ramas**: No trabajes directamente en `main`. Crea una rama descriptiva:
   ```bash
   git checkout -b feature/nombre-de-tu-mejora
   ```
2. **Commits**: Usa mensajes claros y en español (ej: `feat: añadir limpieza de datos`, `fix: corregir error en modelo`).
3. **Pull Requests**: Cuando termines, sube tu rama y abre un PR en GitHub para revisión.

## 📁 Estructura de Proyectos

Cada proyecto debe seguir la estructura estándar:
- `data/`: Datos (no subir archivos >50MB).
- `notebooks/`: Experimentación.
- `src/`: Código productivo.

## 🧪 Requisitos

Asegúrate de actualizar el archivo `requirements.txt` si instalas nuevas librerías.
